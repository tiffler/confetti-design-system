## Notes

**H6 and body primary are both 16px.** That is deliberate — at the bottom of a heading
ramp, size stops being the differentiator and the display face and weight carry it.
Reach for `h6` when it is structurally a heading, `body` when it is prose.

**Display size is container-query driven** — `clamp(36px, 13cqw, 104px)`. The `cqw`
unit needs an ancestor with `container-type: inline-size`; the specimen above sets
it on itself, so the headline scales with the docs panel rather than the viewport.

**The font families here declare system fallbacks** — Fredoka and JetBrains Mono need
to be loaded by the consuming app (or added as a webfont in
`.storybook/preview-head.html`) to render exactly as designed.
